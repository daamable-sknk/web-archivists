# 함께한 사람들 작성 가이드

교육이 끝날 때마다 참가자의 자기소개 웹(또는 과제 URL)을 사이트에 반영하는 방법을 정리한 문서입니다.

## 페이지 위치

| 항목 | 경로 |
|------|------|
| 페이지 | `participants/index.html` |
| 메인에서 링크 | `index.html` 하단 「함께한 사람들 →」 |
| 스타일 | `css/style.css` — `.card-catalog`, `.index-card`, `.index-card.no-link` |

## 기본 원칙

### 1. 누적 — 기존 목록을 지우지 않는다

- **이전 기수 참가자는 그대로 둔다.**
- 새 기수 참가자만 **추가**한다.
- 전체 목록을 새 기수로 **교체하지 않는다.**

### 2. 정렬 — 전체 가나다순

- 성+이름 기준 **한글 가나다순**으로 한 목록을 정렬한다.
- 새 사람을 맨 아래에 붙이지 않는다. 추가·수정 후 **전체를 다시 정렬**한다.
- 같은 성(예: 김) 안에서는 이름 첫 글자 음순으로 배치한다.  
  예: 김고운비 → … → 김진선 → 김태엽 → 김혜수 → 김혜윤

### 3. 누구를 넣을지

| 포함 | 제외 |
|------|------|
| 해당 기수에 **실제로 참여**한 수강생 | 미참여·중도 포기 (명단에 올리지 않음) |
| 과제·자기소개 URL을 공유한 사람 (링크 카드) | **강사·운영** (예: 장대환) |
| 링크 없이 참여만 확인된 사람 (`no-link` 카드) | 이미 목록에 있는 사람을 **중복 추가**하지 않음 |

### 4. 링크 URL

- 가능하면 **마지막 주(또는 최종) 과제** 주소를 사용한다.
- Neocities가 일반적이나, Notion·블로그 등 공유된 최종 URL도 사용할 수 있다.
- 경로에 공백이 있으면 URL 인코딩을 그대로 쓴다.  
  예: `https://pokemsrw.neocities.org/self%20introduction`
- 임시·1주차만 있는 링크보다 **완성된 자기소개 페이지**를 우선한다.

### 5. 키워드

- 교육 **첫날 채팅 자기소개**에 나온 단어·구를 쉼표로 이어 쓴다.
- 카드마다 **4~5개** 정도가 일반적이다.
- 예: `아키비스트, 문화예술, 공공기관, 전문요원, 30대`

## 카드 두 가지 형태

### 링크 있음 (클릭 가능)

```html
<a href="https://example.neocities.org/" target="_blank" rel="noopener noreferrer" class="index-card">
    <span class="card-label">홍길동</span>
    <span class="card-keywords">키워드1, 키워드2, 키워드3, 키워드4</span>
</a>
```

### 링크 없음 (현승인·손정미와 동일)

과제 URL이 아직 없거나, 의도적으로 링크를 두지 않을 때 사용한다. 회색 톤, 호버 없음.

```html
<span class="index-card no-link">
    <span class="card-label">홍길동</span>
    <span class="card-keywords">키워드1, 키워드2, 키워드3, 키워드4</span>
</span>
```

## 작업 절차 (기수 종료 후)

1. **채팅·과제 스레드**에서 참가자 이름, 키워드, 최종 URL을 정리한다.
2. `participants/index.html`의 `.card-catalog` 안 **기존 카드는 삭제하지 않는지** 확인한다.
3. 새 참가자 카드 HTML을 작성한다 (중복 이름 검색 후 추가).
4. 기존 참가자 **링크만 바뀐 경우** 해당 `<a href="...">`만 수정한다.
5. **전체 목록을 가나다순**으로 재배치한다.
6. 사이드바 푸터 `Last updated: YYYY.MM.DD`를 오늘 날짜로 갱신한다.
7. 브라우저에서 `participants/index.html`을 열어 카드 순서·링크·`no-link` 스타일을 확인한다.

## 체크리스트

- [ ] 기존 기수 인원이 그대로 있는가
- [ ] 새 기수만 추가했는가 (중복 없음)
- [ ] 강사·미참여자가 빠져 있는가
- [ ] 전체 가나다순인가
- [ ] 최종 과제 URL을 썼는가
- [ ] `target="_blank" rel="noopener noreferrer"`가 링크 카드에 있는가
- [ ] `Last updated` 날짜를 갱신했는가

## 자주 하는 실수

| 실수 | 올바른 처리 |
|------|-------------|
| 새 기수로 목록 전체 교체 | 기존 카드 유지 + 새 카드만 추가 |
| 새 사람만 맨 아래 추가 | 추가 후 전체 가나다순 정렬 |
| 이미 있던 사람을 또 추가 | 기존 카드 유지, URL만 필요 시 수정 |
| 강사를 참가자처럼 등록 | 운영진은 목록에 넣지 않음 |
| 1주차 임시 링크만 사용 | 최종 Neocities(등) URL로 연결 |

## 참고: 정보 수집 출처

- **이름·키워드**: 수업 첫날 단톡/채팅 자기소개
- **URL**: 주차별 과제 제출 링크 중 **마지막 주** 공유 링크 우선
- 링크 미제출 참가자: `no-link` 카드로 이름·키워드만 표시
