### 구현 내용

#### 1. 레이아웃 구조 (좌우 분할)
**왼쪽 사이드바 (400px, 고정)**
- 사이트 제목 + 영문 부제
- Philip K. Dick 레퍼런스 박스 (회색 배경, 왼쪽 검은 테두리)
- 정보 섹션들 (일정/대상/목표/준비물/장소)
- Apply Now 버튼
- 푸터 크레딧 (새로운 질서 언급)

**오른쪽 메인 콘텐츠 (유동적)**
- 히어로 질문 2개 (큰 이탤릭체)
- 프로그램 소개 텍스트
- 4주 스케줄 (그리드 카드 레이아웃)
- Philosophy 섹션 (중앙 정렬)

#### 2. 디자인 특징
- **컴팩트한 간격**: 섹션 padding 8-10rem → 2-4rem으로 축소
- **밀도 높은 정보 배치**: 스크롤 최소화
- **카드 인터랙션**: 스케줄 카드 호버 시 그림자 + 이동 효과
- **사이드바 배경**: #fafafa (미묘한 회색)
- **메인 배경**: #ffffff (순백)

#### 3. 반응형 대응
- **1024px 이하**: 사이드바가 상단으로 이동 (세로 배치)
- **640px 이하**: 타이포그래피 크기 조정

#### 4. 파일 구조
```
web-archivists/
├── index.html          # 새 원페이지 레이아웃 (현재 버전)
├── index-v1.html       # 첫 번째 버전 백업
├── index-v2.html       # 두 번째 버전 백업 (세로 스크롤)
├── css/
│   ├── style.css       # 새 스타일시트 (현재 버전)
│   ├── style-v1.css    # 첫 번째 버전 백업
│   └── style-v2.css    # 두 번째 버전 백업
├── curriculum/
│   └── index.html      # 커리큘럼 페이지 (업데이트 필요)
└── ...
```

2. **고정 헤더 제거**: 사용자 요청으로 GNB 네비게이션 제거

   - body 배경에 은은한 수직 그리드 라인 (#f8f8f8)
   - 섹션별 교차 배경색 (#fafafa / #ffffff)
   - 레이블 텍스트 앞 "—" 장식
   - 레퍼런스 마크 옆 작은 점

### 주요 스타일
- **배경**: 흰색 베이스 + 미묘한 그리드
- **간격**: 섹션 간 8-10rem padding
- **테두리**: #e0e0e0 (부드러운 회색)
- **스케줄 호버**: 검정/흰색 반전

→ **이 버전은 스크롤이 너무 길다는 피드백으로 원페이지 레이아웃으로 전환됨**

## 2026.01.24 - 콘텐츠 정리 및 최종 구조 확정

### 주요 변경사항

#### 1. 메인 페이지 (index.html) 콘텐츠 개선
**히어로/인트로 섹션 통합**
- 기존: 히어로 섹션(큰 이탤릭 질문) + 별도 인트로 섹션
- 변경: 단일 인트로 섹션으로 통합
  - 질문 2개: 기울임체, 따옴표 유지, 인트로 텍스트와 동일한 크기(1.125rem)
  - `<br><br>` 태그로 질문과 본문 사이 명확한 공백 생성
  - 본문: 2문장 → 4문장으로 확장하여 프로그램 설명 강화

**인트로 텍스트 확장 (2문장 → 4문장)**
```
『아키비스트는 웹을 꿈꾸는가?』는 기록학을 전공했지만 웹에는 아직 익숙하지 않은 실무자들을 위한 재교육 프로그램입니다. 
4주간의 여정 동안 우리는 아카이브와 웹의 경계를 탐구하고, 저장과 연결의 관계를 재발견하며, 협업이 만들어내는 새로운 가능성을 경험합니다. 
HTML과 CSS를 직접 다루며 작은 웹사이트를 손으로 만들어봅니다. 
이 과정을 통해 디지털 환경에서 아카이브가 어떻게 완성될 수 있는지, 그 가능성과 한계를 함께 체험합니다.
```

**레이아웃 조정**
- 인트로 텍스트 `max-width`: 800px → 1000px (커리큘럼 카드 너비와 조화)
- 구분선 위치 변경: 인트로 섹션 상단 → 인트로/커리큘럼 섹션 사이로 이동
- Philosophy 섹션 삭제 (불필요한 하단 콘텐츠 제거)

**사이드바 간소화**
- Philip K. Dick 레퍼런스 박스 삭제 (정보 과다)

#### 2. 커리큘럼 페이지 (curriculum/index.html) 업데이트
**Google Presentation 링크 연결**
- start.md에서 9개 교육 자료의 Google Presentation URL 추출
- 모든 링크를 `/edit` → `/preview`로 변경 (보기 전용)
- Week 02, Week 03 양쪽에 "02-99 코드로 다시 작성하는 자기소개" 공통 배치

**인트로 메시지 교체**
- 기존: "등록하신 수강생 여러분 환영합니다. 아래 링크를 통해..."
- 변경: 강의 자료에서 발췌한 인용문
  ```
  "웹이라는 물리적 공간에서, 웹의 가장 오래된 언어로 표현하는, 나만 만들 수 있는 아카이브"
  ```
- 스타일: 기울임체, 따옴표, 색상 강조 (#333333)
- 출처: `class/01-4 웹에 기반한 아카이브.txt` 321-323줄

**링크 구조**
```
Week 01 (5개)
- 01-1 오늘부터 우리는
- 01-2 디지털로 하는 기록관리
- 01-3 아카이브로서의 웹
- 01-4 웹에 기반한 아카이브
- 01-5 글쓰기 : 자기소개

Week 02 (3개)
- 02-1 개발 환경 만들기
- 02-2 HTML
- 02-99 코드로 다시 작성하는 자기소개

Week 03 (2개)
- 02-3 CSS
- 02-99 코드로 다시 작성하는 자기소개

Week 04 (1개)
- 04 최종 발표 자료
```

#### 3. CSS 개선사항
**인트로 섹션 스타일**
```css
.intro-text {
    font-size: 1.125rem;
    line-height: 1.7;
    max-width: 1000px;  /* 800px에서 확장 */
}

.intro-question {
    font-style: italic;  /* 질문만 기울임 */
}
```

**커리큘럼 페이지 인용문**
```css
.intro-quote {
    font-style: italic;
    color: #333333;  /* 기본보다 진하게 */
}
```

### 콘텐츠 철학 및 디자인 결정
1. **간결함**: 불필요한 장식적 요소 제거 (Philip K. Dick 박스, Philosophy 섹션)
2. **명확성**: 질문과 답변의 흐름이 자연스럽게 이어지도록 구조화
3. **실용성**: 커리큘럼 자료에 직접 접근 가능한 실제 링크 연결
4. **철학 반영**: 강의 자료의 핵심 메시지를 인용문으로 활용
5. **가독성**: 텍스트 너비와 간격 최적화

### 기술적 세부사항
- 강의 자료 텍스트 변환: `.pptx` → `.txt` (class 폴더)
- 모든 외부 링크: `target="_blank" rel="noopener noreferrer"`
- 한글 텍스트: `word-break: keep-all` + `overflow-wrap: break-word`
- 간격 조정: `<br>` 태그 활용 (CSS보다 직관적)
